# Contributing

Thanks for taking the time to contribute.

## Good contribution fit

The best external contributions are usually:

- bug fixes with a small, focused scope
- README clarifications that remove real ambiguity
- implementation improvements with concrete reasoning
- test additions for edge cases or regressions

## Before opening a PR

- keep the scope tight
- explain the problem being fixed
- include focused notes or tests when they materially clarify behavior
- avoid mixing unrelated cleanup with the main change

## Discussion

For larger changes, opening an issue first is usually the fastest way to align on scope.
